# GitHub Copilot Demo Repo

Bu repo, kod yazmadan surec ekiplerinin GitHub Copilot ile nasil daha hizli ve kaliteli analiz uretebilecegini gostermek icin hazirlandi.

## Demo Temasi
Tedarikci Onboarding ve Uyum Sureci Iyilestirme

## Hedefler
- Daginik surec bilgisini net ve olculebilir surec dokumanina donusturmek
- Risk kayitlarini standardize edip onceliklendirmek
- Ham notlardan kaliteli GitHub issue uretmek
- UAT ve kabul kriterlerini hizli olusturmak

## Demo Kapsami
- Surec analizi ve dokumantasyon
- Risk analizi ve kontrol onerileri
- Issue kalitesi iyilestirme
- Test senaryosu ve kabul kriterleri

## Repo Yapisi
- docs/process-overview.md: Baslangic durumu, KPI, darbozazlar
- docs/as-is-to-be.md: AS-IS ve TO-BE modeli
- risk/risk-register.csv: Ornek risk kayitlari
- issues/seed-issues.md: Ham issue adaylari
- tests/uat-scenarios.md: UAT senaryo seti
- prompts/demo-prompts.md: Canli demo prompt kutuphanesi
- templates/output-template.md: Standart Copilot cikti formati
- runbook/demo-runbook.md: 90 dakikalik uygulama akisi
- metrics/demo-kpi-tracker.csv: Demo performans olcum tablosu

## Hizli Baslangic
1. Copilot Chat acin.
2. docs/process-overview.md dosyasini referans alip analiz promptu calistirin.
3. risk/risk-register.csv uzerinde kritik risk cikarimi yaptirin.
4. issues/seed-issues.md listesinden issue standardizasyonu yaptirin.
5. tests/uat-scenarios.md icin yeni edge-case senaryolari urettirin.

## Beklenen Cikti
- Netlestirilmis surec akisi
- Onceliklendirilmis risk listesi
- Kabul kriterleri net issue backlog
- UAT ve kontrol checklist seti

## Not
Bu repo bir egitim ve demo setidir. Gercek sureclerde kurum politikalari ve resmi onay mekanizmalari esas alinmalidir.
